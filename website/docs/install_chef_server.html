Installing a Chef Server 12 on vCloud Air



Create an Ubuntu Server on vCloud Air using vca-cli

If you don't aleady have an Ubuntu Server follow these steps to create one with 2 cpu 4gb ram, 16 GB disk.
You will need to setup ssh keys so that the ubuntu user can login without specifying a password.

Create an Ubuntu Server using vca-cli


Login to the ubuntu server using the ubuntu user.
Then follow the following steps to install the Chef Server

Install the Chef Server 12 package 

su - root
cd /tmp
wget https://web-dl.packagecloud.io/chef/stable/packages/ubuntu/precise/chef-server-core_12.0.7-1_amd64.deb

dpkg -i /tmp/chef-server-core_12.0.7-1_amd64.deb

// confirm the package was added, should be 12.0.7-1 or greater
dpkg -l | grep chef



Create a chef-server.rb override file 

Using your favorite text editor create a file named chef-server.rb
In this file you will specify the fully qualified domain name and port that your chef server will listen on.
A install configuration gets generated using these values so choose carefully, changing them afterwords is a bit tedious.

Change the api_fqdn and Port below

vi /etc/opscode/chef-server.rb

api_fqdn "chef.vcair.us"
nginx['non_ssl_port'] = 8200


Configure the Server

Run the command below, it may run for 5 mins or so.

chef-server-ctl reconfigure


Create a new Organization and a new User

These steps create a new organization with the Chef Server and a new user in the Organization
Two sets of RSA keys are created.  The public keys will be kept on the server.
You will need to safely save the private keys on your workstation or in some other safe location.
If at some point you lose the keys they can be regenerated using the Chef UI once the server is running.
But then all existing chef clients must be updated with the keys.

chef-server-ctl user-create ubuntu Developer youremail@gmail.com welcome1 --filename /home/ubuntu/.ssh/chef-ubuntu.pem

chef-server-ctl org-create chef Chef Software, Inc. --association_user ubuntu --filename /home/ubuntu/.ssh/chef-validator.pem

chown ubuntu ~ubuntu/.ssh/chef-ubuntu.pem 
chown ubuntu ~ubuntu/.ssh/chef-validator.pem

chgrp ubuntu ~ubuntu/.ssh/chef-ubuntu.pem 
chgrp ubuntu ~ubuntu/.ssh/chef-validator.pem


Create the Chef UI 

These commands generate the Chef Server UI and reconfigure the server to include it.
You must have complete the user and org steps above first, before running this command.

chef-server-ctl install opscode-manage
chef-server-ctl reconfigure
opscode-manage-ctl reconfigure



Visit the Chef Server in a browser

These commands generate the Chef Server UI and reconfigure the server to include it.
You must have complete the user and org steps above first, before running this command.

Based on the values you entered in the chef-server.rb file you can now visit the server in a browser.

https://devops.vcair.us:8200





Run Automated tests for the Server to validate installation  



$ sudo chef-server-ctl status

$ sudo chef-server-ctl test




Configuring the vCloud Air Gateway


The vCloud Air Gateway must be configured to allow access to the Chef Server from outside vCloud Air

You need to add at minimum Two DNAT rules and and Two SNAT rules.
Use the vCloud Air Portal to add rules for your Chef Server.

For example Assuming you have created an Ubuntu Server on 192.168.109.2
And you have an available public IP on the Gateway  23.92.225.177
Add The following NAT Rules


d2p3-ext                      SNAT    192.168.109.0/24    Any    23.92.225.177 443   ANY
M9330009684-4424    SNAT    192.168.109.0/24    Any    23.92.225.177  443  ANY

d2p3-ext                      DNAT     23.92.225.177        443    192.168.109.2  443  TCP
M9330009684-4424    DNAT     23.92.225.177        443    192.168.109.2  443  TCP


HAIRPIN NAT CONFIGURATION

Note that a rule must be applied to both the internal and external networks.

If rules are not applied to both networks, then VMs running the Chef Client in the Virtual Data Center will not
be able to connect to the Chef Server, requests will fail with a "Connection Refused" Error.

For a thorough explanation refer to this article  http://www.websterx.com/bwb/vcair/vcloud-air-hairpin-nat/
