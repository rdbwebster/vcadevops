SB2 Scenario

This scenario demonstrates an continuous integration build pipeline using GitHub, Jenkins, Chef and testing with Selenium.
A Spring Boot Form Submission Application is built and deployed whenever a developer modifies the code in the GitHub Project.
The App is deployed to a server that is configured by Chef
Automated testing is performed by Seleniume

The sequence is as follows...

  1. The developer checks in a modification to GitHub
  2. GitHub notifies Jenkins of the change
  3. Jenkins retrieves the code from Github
  4. Jenkins instructs Gradle to build the project
  5. Jenkins instructs Chef to provision the VM for the application
  6. Jenkins deploys the new version of the Application
  7. Jenkins instructs Selenium to perform a test of the App
  8. The developer accesses the new version of the App
